# TestFlight 内部テストプラン

審査提出前に TestFlight で必ず踏む手順。**実機必須**（HealthKit / IAP は
シミュレータでは完全には検証できない）。

---

## 0. テスト端末マトリクス

| 端末 | iOS | 目的 |
|------|-----|------|
| iPhone 16 Pro | 18.x | 最新デバイス・6.7" スクショ |
| iPhone SE (3rd) | 18.x | 4.7" 小画面・古い筐体の UI 崩れ確認 |
| iPad（任意） | 18.x | iPhone専用なので起動NGを確認 |

最低 1 台あれば提出可能だが、SE と Pro の 2 台で当たれば安全。

---

## 1. 初回起動フロー（最優先）

### 権限ダイアログ 3 連発
- [ ] HealthKit 権限ダイアログが**日本語**で表示される
  - 歩数・ワークアウト・睡眠分析の 3 項目が並ぶ
  - 説明文が `Info.plist` の `NSHealthShareUsageDescription` と一致する
- [ ] モーション&フィットネス権限ダイアログが表示される
- [ ] 通知権限ダイアログが表示される（拒否でもアプリが動く）

### 初回オンボーディング
- [ ] 「すべて拒否」でも**クラッシュせずに**ホーム画面に到達する
- [ ] 「すべて許可」で歩数が即座に取得される（今日の CP > 0）

### データ初期化
- [ ] アンインストール→再インストールで完全に初期状態に戻る
- [ ] 街が Lv1 / 累計 CP = 0 / マップサイズ = small (20×20)

---

## 2. HealthKit 統合（実機のみ検証可能）

### 歩数連携
- [ ] iPhone を持って歩く → 数分以内にアプリ内の歩数が更新される
- [ ] アプリをバックグラウンドに置いて再度開く → 最新歩数が反映される
- [ ] Apple ヘルスケアアプリで歩数を手動追加 → アプリに反映される

### 睡眠連携
- [ ] 起動時に前夜の睡眠データが取得される（AutoSleep/Apple Watch 使用時）
- [ ] 睡眠データが 0 のときエラーハンドリングされる

### ワークアウト
- [ ] Apple Watch でワークアウト記録 → アプリの運動 CP に反映
- [ ] ワークアウト種別（ランニング/筋トレ/ヨガ等）が正しく分類される

---

## 3. 5 軸の記録フロー

各軸で次を確認:

### 運動（自動）
- [ ] 歩数から CP が計算される（1 万歩=60CP, 8000 歩=48CP 等）
- [ ] ワークアウト時間が加算される（30 分=40CP）

### 食事
- [ ] 3 食チェック + 間食なし → 100 CP
- [ ] 部分チェック → 適切に按分される
- [ ] 編集後に保存 → 即座に街に反映

### 飲酒
- [ ] 0 杯 → 100 CP
- [ ] 適量 (1-4 杯) → 60 CP
- [ ] 過飲 (5 杯以上) → 0 CP + **居酒屋/廃墟ビル出現**
- [ ] 翌日に過飲しなければペナルティ建物が消える

### 睡眠
- [ ] 7-9 時間 → 100 CP
- [ ] 6 時間 → 60 CP
- [ ] 5 時間以下 → 20 CP
- [ ] HealthKit 取得成功時は手入力が上書きされない

### 生活習慣
- [ ] 水 8 杯 → 30 CP
- [ ] ストレス管理チェック → 40 CP
- [ ] 習慣達成 → 30 CP

## 4. ゲーム側の演出（Phase A/B 後に強化）

### CP 加算時
- [ ] リングパルスが画面中央に出る
- [ ] `+N CP` フローティングテキスト + スパークル
- [ ] タップ軽ハプティック

### レベルアップ時
- [ ] 街レベル上昇時に金色全画面フラッシュ
- [ ] Success ハプティック
- [ ] 建物 Lv アップ時にリングパルス + スパークルバースト + levelUpBurst

### 天気変化
- [ ] CP 400 超 → 晴れ
- [ ] CP 100 未満 → 嵐 + 雷光フラッシュ + 警告ハプティック
- [ ] 曇り・部分曇りで雲レイヤーが流れる

### 時間帯
- [ ] 朝 (7-9) 柔らかい光 / 昼 (10-15) クリア / 夕焼け (16-19) / 夜 (20-4) 群青
- [ ] 夜間に建物の窓ライトが点滅する
- [ ] B029 居酒屋はオレンジの窓、B030 廃墟ビルは窓無し

### マップ拡張
- [ ] 累計 5,000 CP 到達 → 30×30 に拡張 + 金色スパークル
- [ ] 累計 15,000 CP → 40×40
- [ ] 累計 30,000 CP → 50×50
- [ ] 拡張後も既存建物が消えない

## 5. 建物建設

- [ ] 必要 CP 未達の建物は建設ボタンがグレーアウト
- [ ] 未達状態でタップ → error ハプティック
- [ ] 建設成功 → 粉塵リング + スパークル + constructionLanding ハプティック
- [ ] 建設済み建物は再建設不可（カタログで「建設済み」表示）
- [ ] 建物タップ → 情報パネルが出る（名前・Lv・CP・説明文）
- [ ] 建物が時間経過で Lv アップする（XP 蓄積）

## 6. In-App Purchase（Sandbox 環境）

### Sandbox アカウントで検証
- [ ] TestFlight ビルドで Sandbox アカウントに自動サインインする
- [ ] プレミアム購入ボタンをタップ → Apple ID 認証ダイアログ
- [ ] 購入成功 → プレミアム機能がロック解除される
- [ ] アプリ再起動 → プレミアム状態が維持される
- [ ] 「購入を復元」ボタン → プレミアム状態が復元される
- [ ] 購入済みの状態で再度購入試行 → 「すでに購入済み」が表示
- [ ] ファミリー共有: 別の Sandbox アカウントでも復元できる

### プレミアム機能
- [ ] 無料版: マップ拡張が 50×50 で打ち止め → プレミアム購入促進
- [ ] 無料版: 統計画面は 90 日までスクロール可、それ以前は灰色表示
- [ ] 無料版: ウィジェットは小サイズのみ
- [ ] プレミアム版: 上記すべて解放される

- [ ] 大 (プレミアム): 街のプレビュー画像 + 統計サマリ

## 9. エッジケース

- [ ] 機内モード中に起動 → オフラインでも動作（本アプリはネット不要）
- [ ] 低電力モード → アニメーションが軽量化される、機能は維持
- [ ] ストレージ残量少（< 100 MB）→ アーカイブ案内が出る
- [ ] 日付変更（深夜 0 時を跨ぐ）→ 新しい DailyRecord が自動作成される
- [ ] タイムゾーン変更（海外旅行想定）→ 記録が重複しない
- [ ] HealthKit 権限を後から拒否 → アプリが CP 取得にフォールバックする
