# 06 — 品質チェック & 配信ワークフロー

PixelLab.ai で生成した画像を**採用するか捨てるか**の判断基準と、
来週配信までのタイムライン。

---

## A. 採用基準（全アセット共通 9 項目）

生成画像を `Assets.xcassets` に入れる前に**必ず**チェック:

1. [ ] **背景が完全透明**（PNG を Finder のプレビューで開いて確認）
2. [ ] **アイソメ 2:1 比率** が崩れていない（屋根が正面向きになっていない）
3. [ ] **光源が左上**、影が右下（向きが反対なら不採用）
4. [ ] **アンチエイリアスなし**（ソフトエッジのピクセルが無い）
5. [ ] **軸色が識別できる**（緑なら緑、青なら青と一目でわかる）
6. [ ] **他の採用済み建物と並べて浮かない**（色調・ディテール密度が揃う）
7. [ ] **判読可能なテキストが無い**（商標・著作権リスク回避）
8. [ ] **人物・著名人・ブランドロゴが無い**
9. [ ] **32×32 にダウンサンプルしても識別可能**（ミニマップ表示サイズ）

1 つでも NG → 再生成、または手動修正（1 ピクセル単位の修正は Aseprite や
Pixelmator でも可能）

---

## B. 5 軸カラーテスト

生成した建物画像を **軸ごとにグループ化**して並べ、次を確認:

| 軸 | 色の一貫性チェック |
|----|---------------------|
| 運動（緑） | B001-B006 の緑がすべて同系統（ネオン緑と濃緑が混在しない） |
| 食事（橙） | B007-B012 の橙がすべて暖色系（赤 or 黄色寄りが混在しない） |
| 飲酒（紫） | B013-B016 の紫がラベンダー系で統一 |
| 睡眠（青） | B017-B022（B019 除く）の青が水色〜コバルト範囲で統一 |
| 生活習慣（ピンク） | B019, B023-B028 のピンクが暖色ピンクで統一 |

ずれている建物は**再生成 or Photoshop 色相調整**（Hue ±10 以内）。

## D. Xcode 投入手順（再掲）

1. `Resources/Assets.xcassets/PixelArt/Buildings/_Template.imageset` を右クリック → **Duplicate**
2. リネーム: 例 `bld_B025_lv1`（`.imageset` 拡張子は触らない）
3. Xcode のメディアライブラリで選択 → **1x スロットに PNG をドラッグ**
4. 属性インスペクタで:
   - **Render As**: Original Image
   - **Resizing**: 無効（デフォルトでよい）
   - **Preserves Vector Representation**: OFF
5. 実機/シミュレータでビルド → 該当建物が画像化されているか確認
6. もし差し替わらない:
   - `PixelArtRenderer.invalidateCache()` をアプリ起動直後に 1 回呼ぶ
   - Xcode → Product → Clean Build Folder → 再ビルド

## E. 実機スクショ撮影プラン

App Store 審査で各デバイスサイズ 5 枚必要。各スクショのテーマ:

1. **ホーム画面**（街と HUD 全景）
2. **記録ダッシュボード**（5 軸リング + 全軸完了の紙吹雪）
3. **建物詳細**（市庁舎 Lv3 を選択した状態、名前+説明パネル）
4. **建設ショップ**（カタログ画面で複数建物が見える）
5. **統計グラフ**（Swift Charts で週別 CP 推移）

### サイズ

| デバイス | 解像度 |
|---------|--------|
| iPhone 6.7" | 1290×2796（必須） |
| iPhone 6.5" | 1242×2688（必須） |
| iPhone 5.5" | 1242×2208 |
| iPad 13" | 2048×2732（iPad サポート時） |
| iPad 12.9" | 2048×2732 |

iPhone 16 Pro Max シミュレータで撮ると 6.7" サイズが得られる。

## F. 想定される App Store 審査リジェクト理由 & 対策

| リジェクト理由 | 対策 |
|--------------|------|
| ヘルスケアデータ取得目的が不明確 | Info.plist の `NSHealthShareUsageDescription` を日英両方で明記 |
| プレミアム課金の機能説明不足 | App Review Information に「買い切り、復元あり」と明記 |
| 子ども向け UI に見える（ピクセルアート） | レーティングを **12+** に設定（飲酒の記録があるため） |
| 飲酒軸がアルコール販売と誤解 | 説明文に「記録のみ・販売なし」と明記 |
| スクリーンショットにテキストが入っている | VITA CITY の日本語 UI は OK、英語版も用意するとなお良い |

## G. 配信後の改善ループ

1. **週次で 5〜10 枚**ずつ建物 Lv2/Lv3 画像を追加
2. ユーザーの街レベル分布を分析 → 伸びているレベル帯の画像を優先
3. イベント建物（桜祭り、クリスマスなど）を季節追加

画像は Lv ごとに完全一致 or プロシージャル生成にフォールバックする設計
（低 Lv 画像の流用はしない）なので、**「Lv1 から順に完成させて埋めていく」**
運用が本プロジェクトに最適。未生成 Lv はプロシージャル版が代わりに表示される。
